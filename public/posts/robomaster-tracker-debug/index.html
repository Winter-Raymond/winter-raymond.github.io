背景#

RMUC2026后，尝试解决一些比赛过程中发现的问题，主要是定位到了手眼标定，原来的手眼标定和机械给的参数差得蛮多的，而且自己的静态误差也大得离谱，估计距离8m的情况下，观察高度为0.86m的目标，观测高度为1.1～1.2m，很要命的误差，后来借鉴了小鱼的标定手法，主要参考这一篇文章，但是实际测试下来效果，怎么说呢，对比之前的超绝误差有很大进步，但在8m的距离下依旧不够用

火控不符合实际#

现象#

我们英雄打击前哨站的策略是定头，就是枪管始终指向机器人中心，等待装甲板转过来（因为电机劲儿太小控不住云台），所以按理来说弹丸落点就算高度打不准，也应该一直打在机器人中心位置，但实际就是弹丸落点很散，有时左偏有时右偏，整体效果很难绷。

排查#

最开始觉得是擦枪管的问题，就用粉笔在枪管里面画了一小圈圆，如果擦的话粉笔会被擦掉，后来确实有这个问题，但是似乎不是根源，于是接着排查。 后面又觉得是观测问题，比如镜头上有指纹之类的，后来看了一下确实有，擦一下就好了。哦，还有超绝坑货，之前为了保护那个镜头加的一个透明的亚克力板（感觉材质应该是），我真服了，那个板都损坏了，我透过那个板子看前哨站都模糊，艹，赶紧卸下来，以后也别加了 用 plotjuggler 看了曲线，大概就是 /processor/target 也就是过了卡尔曼滤波的数据，应该都是目标相对于弹丸发射点的位置或者速度，测量了一下，发现距离上也没什么问题，实际距离可能在 8.2～8.3m，但是观测距离在 8.28m 还算不错，又看了电控的返回值和我们的目标值，发现其实也还好，跟得算不错了，没有稳态误差，后来觉得可能还是手眼标定的问题，因为距离近了就好了，但是如果是手眼标定的问题，为什么在其它条件都相同的情况下会有时候偏左，有时候偏右呢？甚至车也没有重新上电，没有重启代码。百思不得其解，然后就晚上7:30了，要润去上选修课了，上课的时候也在想。 最后想到一个点，之前由于为了表示自己尊重科学，过于 相信 卡尔曼滤波了，直接在前哨站的火控逻辑里加了一个：

    double pred_cx = cur_x + vx * predict_time;
    double pred_cy = cur_y + vy * predict_time;
    double aim_yaw = std::atan2(cur_y, cur_x);

但是，注意一个点，由于卡尔曼滤波只是进行状态估计，所以 vx 和 vy 都只能是一个较低的值，很难是正儿八经的0.0，加上8m的直线距离下，12m/s弹丸飞行时间接近 1s 这直接引入了几个cm量级的误差。 这样既可以解释他为什么左右飘，又可以解释他为什么飘移动是随机的。

解决#

其实也好说，在前哨站逻辑里的vx和vy置零就行了：

    double vx    = msg->velocity.x;
    vx = 0.0;
    double vy    = msg->velocity.y;
    vy = 0.0;

注意有三个函数 processOutpost、processOutpostFallback 和 processOutpostStatic 都得改 别怪我哦，这样先写一下嘛，以后再正儿八经改。

效果#

正反转效果很好，至少yaw方向是正确的，而且只打中心了，说明确实是这个问题。 真要抽自己了

总结#

力竭了，真要燃尽了，又是被抽象bug折磨的n天，艹。下次还是注意点儿工程实现吧，尽力实现贴近工程的实现。
